How does React Do This?
Crazy high level

Source Facebook React Reconciliation

  • From your first main component - data is checked against the previous data the component had.
  • If it's changed, run the component render process - and compare this virtual dom output with the last virtual dom output. (reconciliation)
  • When a change is detected, a DOM changes diff is calculated between both the previous virtual dom and the new calculated virtual dom of the component
  • These DOM change diffs are then applied - Can be as simple as 'element.classList.add('className');
  • Will calculate if DOM nodes can be reused/repurposed, or if it's quicker to simply remove and replace
  • With sub-components, parent containers will always have had a reconciliation difference in data that will trigger the child component.
  • Calculating this every 100ms (against motorsports for example), is trivial for browsers
  • This is not slow, last match-tracker profile proved that, it's DOM manipulations that are slow.
