How does React Do This?
Crazy high level

Source Facebook React Reconciliation

  • From your first component - data is checked against the previous data the component had (reconciliation).
  • If it's changed, run the component render process - and compare this virtual dom with the last virtual dom.
  • When a change is detected, a diff is calculated between both the previous virtual dom and the new calculated virtual dom of the component
  • These diffs are then applied - Can be as simple as 'element.classList.add('className');
  • Will calculate if DOM nodes can be reused/repurposed, or if it's quicker to simply remove and replace
  • With sub-components, parent containers will always have had a reconciliation difference. They may not have had a virtual dom difference
  • Calculating this every 100ms (against motorsports for example), is trivial for browsers
  • Wouldn't it be slow calling all the components render methods?
  • Nope, last match-tracker profile proved that, it's DOM manipulations that are slow.
